# CC-161.1 — Binary contested clarifier must pit ATTITUDE-OPPOSITES, not the two picks

> CC-161 fixed the Ne/Si garbage (good) but deviated from spec on *which* two
> functions to pit. It surfaces `binaryContestedJudgingPair = [thinkingPick,
> feelingPick]` (e.g. `[te, fi]`) and pits **Te vs Fi** — but that doesn't
> discriminate the type. INTJ = Ni-Te-Fi-Se; INFJ = Ni-Fe-Ti-Se: **Te and Fi are
> both in the INTJ stack**, so Te-vs-Fi can't tell INTJ from INFJ. The
> discriminators are the *attitudes*: **Fi-vs-Fe** and **Te-vs-Ti**. CC-161's own
> CC asked for exactly that ("Fi/Fe, Te/Ti"); this corrects the deviation.

## The rule

When a binary judging axis is contested, the clarifier must re-ask the **attitude
of the contested pick(s)** — pit each picked function against its same-dimension
opposite:
- feeling pick `fi` → **Fi-vs-Fe** head-to-head
- feeling pick `fe` → **Fe-vs-Fi**
- thinking pick `te` → **Te-vs-Ti**
- thinking pick `ti` → **Ti-vs-Te**

These are the function pairs that actually flip the type (Te+Fi = INTJ; Ti+Fe =
INFJ). Pitting the two *picks* against each other (Te-vs-Fi) is never the right
comparison — it asks the person to rank functions they already hold.

Same correction for the perceiving axis: contested perceiving pick `ni` →
**Ni-vs-Ne**, `se` → **Se-vs-Si**, etc. — attitude opposites, not the two picks.

## Tasks

**T1.** In `lib/followUpQuestions.ts`, where the binary-contested pair routes to
the head-to-head (CC-161's `binary_contested_judging` / `binary_contested_perceiving`
block), build the head-to-head from **each contested pick vs its same-dimension
attitude-opposite**, not from the `[thinkingPick, feelingPick]` (or
`[Npick, Spick]`) tuple. If both members of the judging axis are in question
(the type genuinely hangs on both attitudes, as in INTJ↔INFJ), it's acceptable to
generate up to two head-to-heads (Fi-vs-Fe AND Te-vs-Ti) — cap per the second-pass
budget; lead with the feeling one (it's the cleaner INTJ/INFJ tell).

**T2.** Keep CC-161's other wins intact: perceiving clarifier still suppressed when
perceiving is settled; no `topPickCountFor` on the binary path; no double-emit
(the `=== undefined` slot gates stay).

**T3 — I/E-neutral, object-based voices (critical — owner-identified confound).**
An exceptionally introverted person leans toward the *introverted-sounding* voice
on every binary, independent of their true function — so a truly-Fe INFJ picks Fi
just because she's introverted, and the clarifier would re-trigger the same error.
The attitude head-to-head voices MUST test the function's **object**, not its
inward/outward flavor:
- **Fi** = "my read calibrates to my own values — what feels true to me."
- **Fe** = "my read calibrates to what the people I'm responsible for need — the
  harmony of the room." (Phrase so an introvert can fully own it; Fe is NOT
  "I'm social/outgoing.")
- **Ti** = "I check whether it holds together by my own internal logic."
- **Te** = "I check it against what works / the external result."
Both options in each pair must be equally ownable by an exceptionally introverted
person — the choice should reflect *what the function attends to*, never the
person's overall temperament.
