# CC-153.1 — Guard the admin roster (and any direct readers) against placeholder / partial constitutions

> Point-release on CC-153. CC-153's partial-import path (T3) stores
> `inner_constitution = {}` when `buildInnerConstitution` throws on a partial
> legacy answer set. That's correct for the re-derive-on-read design — BUT the
> admin sessions roster reads constitution fields directly (not through
> `detectStaleShape`), and two of those reads are unguarded, so a single
> deferred-derivation import turns the whole roster into "Failed to load
> sessions." That breaks exactly the workflow CC-153 enables: import a partial
> legacy test → can't see the roster → can't reset Q-T or mint the gap-fill link.

## Verified root cause

`app/admin/sessions/page.tsx`:
- `deriveTopCompassSignalId(ic)` — `for (const s of ic.signals)` (~L106). `{}.signals`
  is `undefined` → "undefined is not iterable" → throws.
- `countAllocationTensions(ic)` — `ic.tensions.filter(...)` (~L118). `{}.tensions`
  is `undefined` → throws.
- Both run inside `summaries.map(...)` in `loadSessions`, which is wrapped in the
  page's try/catch → the table is replaced by the `loadError` message for ALL
  sessions whenever any row has a placeholder/empty constitution.
- (`ic.lens_stack?.dominant` and `ic.belief_under_tension?.epistemic_posture`
  on the same path are already optional-chained and safe. The
  report/detail render path is also safe — it goes through `detectStaleShape`
  → re-derive before reading fields. The roster is the gap.)

## Execution mode

Single pass, proceed without pausing; flag ambiguity. Defensive-read fix +
placeholder hardening. No schema/engine/derivation-math change.

## Tasks

**T1 — guard the two roster reads.** In `app/admin/sessions/page.tsx`, make
`deriveTopCompassSignalId` and `countAllocationTensions` tolerate a missing/empty
`signals`/`tensions` (e.g. iterate `ic.signals ?? []`, filter `(ic.tensions ?? [])`).
A constitution with no signals/tensions should yield `top_compass = null` and
`allocation_tensions_count = 0`, not an exception. This is the load-bearing fix —
it makes the roster robust to ANY empty/partial constitution, not just imports.

**T2 — harden the import placeholder.** In
`app/api/admin/sessions/import/route.ts`, change the derive-failure placeholder
from `{}` to a minimally-iterable shape — at least
`{ signals: [], tensions: [] } as unknown as InnerConstitution` — so the stored
row is safe for any direct reader, belt-and-suspenders with T1. Keep
`engine_shape_version: null` (that's what still drives re-derive on read).

**T3 — scan for other unguarded direct readers.** Grep for other code paths that
read `inner_constitution` fields directly (not via `detectStaleShape`/re-derive)
and could be hit by a deferred-import row before re-derivation — e.g. any admin
list/aggregate, API summary, or export that touches `.signals` / `.tensions` /
`.lens_stack` / `.drive` without optional-chaining or a guard. Guard any found;
if a path is already safe (optional-chained or re-derives first), note it and
change nothing.

## Allowed to modify

- `app/admin/sessions/page.tsx` (T1)
- `app/api/admin/sessions/import/route.ts` (T2)
- any additional direct-reader file surfaced by T3 — guard only; list them.

Do NOT change the engine/derivation, the portable shape/validator, `saveSession`,
`detectStaleShape`, or the schema.

## Acceptance criteria

1. With a deferred-derivation imported session present (constitution placeholder,
   `engine_shape_version = null`), `/admin/sessions` lists normally — the import
   appears with `—` for Compass-top and `0` for allocations, and every other
   session renders. No "Failed to load sessions."
2. Opening that session's detail page works and the report re-derives (unchanged
   from CC-153 — confirm it still does).
3. T3's scan is reported: which other direct readers exist and whether each was
   already safe or needed a guard.
4. `tsc` + lint clean. No schema/engine/derivation change. Existing roster
   behavior for normal (fresh) sessions is unchanged.
