## Method note

The data flow is already correct — `buildInnerConstitution` populates `goalSoulGive`, `goalSoulMovement`, and `goalSoulPatterns` on the constitution object that gets passed to both the markdown renderer AND the React component. The React component just hasn't been reading those fields. This CODEX surfaces what's already there.

The SVG embedding via `dangerouslySetInnerHTML` is safe because the SVG is generated by our own `lib/goalSoulDashboard.ts` code, never derived from user input. React's standard escape-hatch for embedding pre-rendered HTML/SVG is the correct tool here. Do not over-engineer with React-component SVG translation.

If during implementation you find that the existing component pattern uses something like `<MarkdownRenderer markdown={...} />` for sections, you can fall back to passing the markdown produced by `renderMirror.ts` for these two sections. But the explicit JSX path is preferred for visual consistency with sibling sections.
