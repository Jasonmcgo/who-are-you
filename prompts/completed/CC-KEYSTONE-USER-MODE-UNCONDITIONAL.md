# CC-KEYSTONE-USER-MODE-UNCONDITIONAL

## Objective
Make user-mode Keystone rendering produce a user-mode-correct artifact regardless of whether the LLM rewrite cache hits. Currently, when the cache misses (every live user), the Keystone two-tier branch falls through to the engine field-list ("Likely value:" / "Wording temperature:" / "How you take in new things to revision:" / "Unsure" bullets) plus the engine prose paragraph — exposing engine artifacts at the most emotionally-loaded moment in the report. After this CC, the engine artifacts never appear in user mode, period.

## Sequencing
Lands after CC-LIVE-SESSION-LLM-WIRING (which will reduce cache misses to rare cases). Even with on-demand wiring, the LLM call can fail or time out — the fallback contract must still be user-mode-correct, not engine-prose.

## Scope
The Keystone section emission in lib/renderMirror.ts. Two layers:
- If LLM cache hits (or on-demand resolution succeeds): render verbatim belief quote + LLM-rewritten paragraph. Same as today's "happy path."
- If LLM cache misses AND on-demand resolution fails (or is skipped): render verbatim belief quote + a deterministic minimal fallback paragraph that names the value cluster and the cost surface in one or two sentences. No field-list. No engine valueOpener paragraph. No "Unsure" labels.

## Do not
- Modify clinician mode. Field-list + engine paragraph stays byte-identical for audit/debug.
- Modify the LLM prompt or cache key.
- Touch React surface (KeystoneReflection.tsx) — its disclosure pattern handles the user-mode display already.
- Add new dependencies.
- Change the belief-quote handling (it must remain verbatim).
- Touch any other section.
- Bump synthesis3 / gripTaxonomy / proseRewriteLlm hashes.

## Rules

### 1. User-mode Keystone is unconditionally free of engine artifacts
Regardless of cache state, the rendered user-mode Keystone contains:
- No bullet matching `- **[anything]:**` between the belief quote and the next section heading.
- No occurrence of the literal string "Unsure" anywhere in the Keystone section.
- No engine valueOpener paragraph (the one that currently starts with "Your selections place this belief inside…" or its archetype-equivalent).
- No literal strings "Likely value:" or "Wording temperature:" or any "to revision:" pattern.

### 2. Three-tier fallback
Render path picks the best available content in this order:
- **Tier A** (cache hit): the LLM-rewritten paragraph from keystone-rewrites.json, spliced in after the verbatim belief quote.
- **Tier B** (cache miss, on-demand resolution succeeds): the LLM paragraph just generated by CC-LIVE-SESSION-LLM-WIRING.
- **Tier C** (cache miss and on-demand fails or is skipped): a deterministic minimal fallback paragraph that:
  - Names the value cluster the engine identified (e.g. "Knowledge, Peace, Faith, and Honor").
  - Names the cost surface labels the user marked, if any.
  - Names whether correction channels were marked, if applicable.
  - Reads as plain prose, not engine artifacts.

Example Tier C template (architect-archetype, Jason inputs):
> "Your selections place this belief inside Knowledge, Peace, Faith, and Honor. You marked Close relationships, Money / Wealth, Reputation, and Job / Career as concrete costs you would bear for it — a wide cost surface. The shape this points to is a belief held inside what you protect, not outside it."

This is plain English. No "Likely value" labels. No "Unsure" status. The deterministic fallback exists for the rare case when both cache and on-demand fail; under normal operation, Tier A or Tier B serves the LLM paragraph.

### 3. Clinician mode preserved
The clinician-mode branch renders exactly as today: belief quote, field-list with full audit metadata, engine valueOpener paragraph. Byte-identical to pre-CC baseline.

### 4. Verbatim belief quote in all tiers
All three tiers open with the user's exact belief statement, quoted verbatim.

### 5. Audit checks rendered structure, not literal strings
The audit gate must verify:
- No bullet list immediately following the belief quote (regex check on rendered markdown structure).
- No "Unsure" literal string anywhere in the Keystone section.
- The verbatim belief quote appears as the first content after the section heading + lead-in italic.
- The paragraph below the quote contains at minimum the value cluster names AND either the cost-surface names or a clear "no cost surface marked" phrasing.

Audit must run against multiple input shapes:
- Cohort fixture with cache hit (Tier A).
- Synthetic non-fixture with on-demand success (Tier B, requires CC-LIVE-SESSION-LLM-WIRING).
- Synthetic non-fixture with on-demand disabled (Tier C, deterministic fallback).

## Implementation notes
- The deterministic Tier C template lives in lib/keystoneRewriteLlm.ts or a sibling file. It composes from the same engine inputs (valueDomain, topCompassValueLabels, costSurfaceLabels, etc.) the LLM uses.
- Templates may differ per archetype (architect / caregiver / steward / unmapped) for register-appropriate fallback prose, but each must satisfy Rule 1 absolutely.
- The Tier C fallback is a safety net, not a primary surface. Under normal operation with CC-LIVE-SESSION-LLM-WIRING active, users should rarely see Tier C.

## Audit gates
- Three-tier rendering verified for at least one fixture per archetype:
  - Tier A: cohort fixture with cache hit produces LLM paragraph.
  - Tier B: synthetic non-fixture triggers on-demand and produces LLM paragraph.
  - Tier C: synthetic non-fixture with on-demand disabled produces the deterministic minimal fallback.
- All three tiers: zero `- **[anything]:**` bullets, zero "Unsure" literal strings, zero engine valueOpener paragraphs in user-mode rendered markdown.
- Verbatim belief quote present in user-mode for all three tiers.
- Clinician mode byte-identical to pre-CC baseline across the 21-fixture Keystone cohort.
- 41-fixture sweep stays green.
- tsc + lint clean.
- Cost: $0 (no LLM regen; Tier B uses on-demand from CC-LIVE-SESSION-LLM-WIRING).
