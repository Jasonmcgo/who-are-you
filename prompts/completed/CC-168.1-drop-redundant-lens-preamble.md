## The problem (confirmed on all 4 review fixtures)

CC-168 rewrote the Lens Strength to open with the dom/aux pair:
> **Strength** — A pattern-discernment gift: you read through the pattern-reader and act through the structurer…

But the italic Lens cardHeader directly above it already says the same pair:
> *Your processing pattern leans toward the pattern-reader, supported by the structurer.*

So the same dom/aux pair is restated back-to-back. The Strength sentence now
carries it more gracefully, so the header is redundant — but ONLY on the Lens
card. Every other card's `cardHeader` is distinct content (e.g. Compass:
"When something has to give, you appear to protect Faith, Peace, Honor… first")
and must stay.

## The fix — `lib/renderMirror.ts`, the SWOT card loop (~L1648-1650)

Current default/high-confidence branch emits the header for every card:
```ts
    } else {
      out.push(`*${card.cardHeader}*`);
    }
```

Change it to suppress the header for the LENS card only (its dom/aux read now
lives in the Strength sentence). Keep it for all other cards:
```ts
    } else if (id !== "lens") {
      out.push(`*${card.cardHeader}*`);
    }
    // CC-168.1 — Lens (high-confidence) header suppressed: its
    // "leans toward X, supported by Y" line duplicated the CC-168 Strength
    // sentence ("you read through X and act through Y") back-to-back. The
    // Strength carries the dom/aux read now. Other cards keep their distinct
    // cardHeader; the low-confidence Lens hedge variants below are untouched.
```
