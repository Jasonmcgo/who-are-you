## Why (the locked game design, 2026-05-25)

Jason locked the loop: one Body Card per round, one shared card, room votes,
engine picks, reveal compares. The base +2/+2/+1 (max 5) is correct. Two things
the design needs that CC-175 lacks:
- A player who correctly reads that the engine itself is *torn* between two people
  ("both, but for different reasons") should be rewarded — that argued-over moment
  ("No, Ashley sees the emotional weather; Jason sees the system failure") is the
  product. Needs an engine **split** signal + a **+3** scoring path.
- Round selection should produce *debate*, not just a lopsided pick. A card with a
  clear top but a live runner-up is more fun than a 1.0-vs-0.0 blowout.

## Change 1 — engine "split" signal (`types.ts` + `engine.ts`)

`EnginePick` (currently: playerId, displayName, score, confidence, matchedTags,
reason, runnerUp). Add:
```ts
/** True when the top two players are within SPLIT_EPS — the engine itself
 *  is torn ("both, for different reasons"). Distinct from confidence: a
 *  near-tie at the TOP, not just a small overall spread. */
isSplit: boolean;
```
In `getEnginePick`: `isSplit = runnerUp ? (top.score - runnerUp.score) < SPLIT_EPS : false`.
Set `SPLIT_EPS = 0.03` (tighter than the `low`-confidence floor of 0.06 — a split
is a near dead-heat, not merely "low confidence"). Export the constant. Leave
`confidence` and its 0.12/0.06 bands untouched.

## Change 2 — Both/Multiple scoring (`types.ts` + `scoring.ts`)

`RoundGuessInputs` (currently has `guessedPlayerId: string`). Make the special
guess first-class:
```ts
guessedPlayerId?: string;            // present for a normal player vote
guessedSpecial?: "both" | "nobody";  // present instead, for the special tiles
```
`RoundScoreBreakdown`: add `splitRead: boolean;` and raise the `points` cap
comment to `0..5` still for normal rounds, `0..6` when the split bonus stacks with
a room match (see below).

`calculateCardScores` rules:
- **Normal card** (`enginePick.isSplit === false`): unchanged — +2 engine match,
  +2 room match, +1 perfect-read bonus when guess == engine == room. A
  `guessedSpecial` vote scores 0 on a non-split card (the engine had a clear pick).
- **Split card** (`enginePick.isSplit === true`): a `guessedSpecial === "both"`
  vote is the correct read → **+3** (`splitRead = true`). It may ALSO take the +2
  room match if the room's plurality was likewise "both" (treat "both" as a
  votable option in `getRoomWinner`). No engine-pick match applies (there is no
  single engine pick to match on a split). Max split-round score = 5 (3 + 2).
- `guessedSpecial === "nobody"`: scores 0 in both cases (no engine signal it maps
  to in MVP) — keep it votable but unscored; note this for CC-177.

`getRoomWinner` must count `"both"` as a candidate alongside player ids, so a room
that pluralities on "both" returns a sentinel the scorer recognizes (e.g. return
`"__both__"` or extend the return type). Keep the tie → `undefined` (Identity Fog)
rule.

## Change 3 — selection tuning (`generate.ts`)

Current ranking (per CC-175 spec): `confidenceBoost + enginePick.score −
diversityPenalty`. Extend the per-candidate score with:
- **Runner-up tension**: reward a *present, live* runner-up — e.g. add a term that
  peaks when the top-vs-runner-up gap sits in a "debatable" band (clear top, but
  the runner-up is within ~0.04–0.10), and is lower for both blowouts (gap huge)
  and dead-heats (gap ~0, which now route to split cards). This makes rounds
  argue-able. Keep it a bounded additive term; don't let it dominate engine score.
- **Soft per-player coverage**: across the generated game, try to feature each
  selected player as an engine pick **at least once** before repeating anyone.
  Implement as an escalating bonus for not-yet-featured players (or an escalating
  penalty for repeats) — strong enough to spread targets across 6–8 players, weak
  enough that a strongly-justified repeat still wins. This generalizes CC-175's
  existing `diversityPenalty`; do NOT hard-force a permutation (no rigid
  "everyone exactly once").
